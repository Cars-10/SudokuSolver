<task type="auto">
  <name>Task 2: Implement CP algorithm in V with bitset-based constraint propagation</name>
  <files>Algorithms/CP/V/cp.v, Algorithms/CP/V/runMe.sh</files>
  <action>
Create Algorithms/CP/V/ directory structure. Port CP from C to V:

**cp.v structure**:
- Use `mut grid := [][]int{len: 9, init: []int{len: 9, init: 0}}` for puzzle
- Use `mut candidates := [][]int` for bitsets (int type, bits 1-9 represent digits)
- Implement `propagate(row int, col int, digit int)` to eliminate candidates
- Implement `solve() bool` with backtracking recursion
- Use global `mut iterations = 0` counter, increment BEFORE validity check
- Parse .matrix file from os.args
- Initialize candidates: for empty cells, set to 0x3FE (bits 1-9), for filled cells set to 0

**Bitwise operations**:
- Check if digit available: `(candidates[row][col] & (1 << digit)) != 0`
- Remove candidate: `candidates[row][col] &= ~(1 << digit)`
- Add candidate back: `candidates[row][col] |= (1 << digit)`
- Count candidates: use bitwise operations or iterate bits 1-9

**Critical requirements**:
- V's bitwise operators: & (AND), | (OR), ~ (NOT), << (left shift)
- Must propagate constraints to row, column, and 3x3 box
- Backtracking must restore candidate sets correctly
- Print output format: puzzle, then "Solved in Iterations=67"
- **Avoid**: Don't use map[int]bool for candidates (bitset is faster and matches reference). Don't use complex V features (keep it similar to C).
