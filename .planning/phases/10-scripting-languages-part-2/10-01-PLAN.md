Create Algorithms/CP/PHP/ directory structure. Port the CP algorithm from C to PHP:

**cp.php structure**:
- Use associative array for CPGrid: ['values' => [[...]], 'candidates' => [[...]]] (2D arrays for bitsets)
- Define bitset helper functions:
  - function has_candidate($set, $digit) - test bit
  - function add_candidate($set, $digit) - set bit (returns new value)
  - function remove_candidate($set, $digit) - clear bit (returns new value)
  - function count_candidates($set) - popcount
- Implement function init_grid(&$grid, $puzzle) - set empty cells to candidates = 0x1FF
- Implement function assign(&$grid, $row, $col, $value) - place value and propagate constraints
- Implement function eliminate(&$grid, $row, $col, $candidates_to_remove) - remove candidates and propagate
- Implement function propagate(&$grid) - apply constraint propagation until quiescence
- Implement function find_mrv_cell($grid) - find cell with Minimum Remaining Values (returns [$row, $col] or null)
- Implement function cp_search($grid) - recursive search with propagation
- Implement function parse_matrix($filename) and main execution with global $cp_iterations counter
- Add shebang: #!/usr/bin/env php
- Use strict typing: declare(strict_types=1);

**Critical requirements**:
- Use PHP integers for bitsets (PHP integers are 64-bit, plenty for 9 bits)
- Bitset operations: use bit shifting and masking (1 << $digit, $set & (1 << $digit))
- Print output format: puzzle state, then "Solved in Iterations=67"
- Increment $cp_iterations at start of cp_search
- Pass grid by reference where needed (&$grid) to avoid deep copies
- Custom popcount implementation (PHP doesn't have built-in popcount)

**What to avoid and WHY**:
- Do NOT use PHP arrays for candidates - must use bitsets (integers)
- Do NOT add extra constraint propagation techniques
- Do NOT change MRV selection order
- Do NOT use Composer packages - keep implementation self-contained
