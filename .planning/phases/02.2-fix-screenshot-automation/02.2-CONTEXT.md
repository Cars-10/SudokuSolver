The fix should produce a clean, properly-rendered 1920x1080 screenshot that shows exactly what you'd see in a browser â€” the performance chart and the top 5 scoring languages, framed consistently every time.

<essential>
## What Must Be Nailed

- **Consistent framing** - Every screenshot captures the same elements (chart + top 5) in the same position, every time
- **Proper dimensions** - 1920x1080, not 8000+ pixels
- **Correct rendering** - Screenshot looks like the actual browser view, not a broken/partial render

</essential>

## What's Out of Scope

- Multiple image formats - PNG only, no JPEG/WebP options needed
- Screenshot comparison/diffing - Not detecting visual changes between runs

</boundaries>

<specifics>
## Specific Ideas

- Target content: Chart + Top 5 languages by score table only
- Resolution: Standard desktop 1920x1080
- The current screenshot is creating but rendering incorrectly and oversized

</specifics>
