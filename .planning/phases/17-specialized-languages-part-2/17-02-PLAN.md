  <files>Algorithms/CP/Vala/cp.vala, Algorithms/CP/Vala/runMe.sh</files>
  <action>
Create Algorithms/CP/Vala/ directory structure. Port CP from C to Vala:

**cp.vala structure**:
- Use `int[,] grid = new int[9, 9]` or `int[][] grid` for puzzle (2D array syntax)
- Use `int[,] candidates` for bitsets (int type, bits 1-9 represent digits)
- Implement `void propagate(int row, int col, int digit)` to eliminate candidates
- Implement `bool solve()` with backtracking recursion
- Use static `int iterations = 0` counter, increment BEFORE validity check
- Parse .matrix file from args[1]
- Initialize candidates: for empty cells, set to 0x3FE (bits 1-9), for filled cells set to 0
- Use `void main(string[] args)` as entry point

**Bitwise operations**:
- Check if digit available: `(candidates[row, col] & (1 << digit)) != 0`
- Remove candidate: `candidates[row, col] &= ~(1 << digit)`
- Add candidate back: `candidates[row, col] |= (1 << digit)`
- Count candidates: iterate bits 1-9 and check each

**Critical requirements**:
- Vala's bitwise operators: & (AND), | (OR), ~ (NOT), << (left shift)
- Must propagate constraints to row, column, and 3x3 box
- Backtracking must restore candidate sets correctly
- Print output format: puzzle, then "Solved in Iterations=67"
- **Avoid**: Don't use Gee.HashSet for candidates (bitset is faster and matches reference). Don't use complex OOP patterns (keep it procedural like C).
