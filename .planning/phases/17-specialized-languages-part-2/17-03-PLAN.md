  <name>Task 2: Implement CP algorithm in Wren with bitset-based constraint propagation</name>
  <files>Algorithms/CP/Wren/cp.wren, Algorithms/CP/Wren/runMe.sh</files>
  <action>
Create Algorithms/CP/Wren/ directory structure. Port CP from C to Wren:

**cp.wren structure**:
- Use `var grid = List.filled(9, null).map {|_| List.filled(9, 0) }.toList` for puzzle
- Use nested lists for candidates (Num type for bitsets, bits 1-9 represent digits)
- Implement `propagate(row, col, digit)` method to eliminate candidates
- Implement `solve()` method with backtracking recursion
- Use static field `var iterations = 0` counter, increment BEFORE validity check
- Parse .matrix file from System.arguments[0]
- Initialize candidates: for empty cells, set to 0x3FE (bits 1-9), for filled cells set to 0

**Bitwise operations**:
- Check if digit available: `(candidates[row][col] & (1 << digit)) != 0`
- Remove candidate: `candidates[row][col] = candidates[row][col] & ~(1 << digit)`
- Add candidate back: `candidates[row][col] = candidates[row][col] | (1 << digit)`
- Count candidates: iterate bits 1-9 and check each with bitwise AND

**Critical requirements**:
- Wren's bitwise operators: & (AND), | (OR), ~ (NOT), << (left shift)
- Must propagate constraints to row, column, and 3x3 box
- Backtracking must restore candidate sets correctly
- Print output format: puzzle, then "Solved in Iterations=67"
- **Avoid**: Don't use Map for candidates (bitset is faster and matches reference). Don't use Wren's advanced features (keep it simple).
