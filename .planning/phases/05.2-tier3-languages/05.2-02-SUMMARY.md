## Decisions Made

- Pascal: Must use `Solve()` with parentheses for recursive calls (Free Pascal quirk)
- Prolog: Use dynamic predicates for mutable counter state

## Issues Encountered

### Pascal
- **Major bug**: `Solve` without parentheses was interpreted as checking the Result variable, not as a recursive call
- **Solution**: Changed to explicit `Solve()` function call syntax
- Extensive debugging required to identify this Free Pascal behavior

### Prolog
- Prolog's native backtracking conflicts with explicit iteration counting
- **Solution**: Use `assertz`/`retract` pattern for mutable counter that persists across backtracking
